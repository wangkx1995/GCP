## 已实现能力

- 新增独立库 `crates/remote-file-source`。
- `--input` 继续用于本地文件或目录模式。
- `--source-config source.toml` 用于 FTP/SFTP 远程模式。
- `--input` 与 `--source-config` 互斥。
- 远程模式要求 `--scan-start-time "yyyy-MM-dd HH:mm:ss"`。
- 支持 FTP 连接、登录、递归扫描、正则匹配和流式下载。
- 支持 SFTP 连接、登录、递归扫描、正则匹配和流式下载。
- 远程文件先下载到 `download_dir`，再复用阶段 1 本地解析链路。
- 下载使用 `<文件名>.part` 临时文件，成功后重命名为最终文件。
- 本地已存在同名最终文件时会覆盖并打印日志。
- 支持连接失败重试，默认 `connect_retry = 3`。
- 支持下载失败重试，默认 `download_retry = 3`。
- 支持重试间隔，默认 `retry_interval_secs = 30`。
- 支持连接超时，默认 `connect_timeout_secs = 30`。
- 支持读写超时，默认 `read_timeout_secs = 300`。
- 支持本地缓存清理，默认 `cache_retention_days = 7`。
- 缓存清理只删除 `download_dir` 直接子级普通文件，不递归删除子目录。
- `.part` 临时文件作为普通文件参与缓存清理。
- 支持 `${SCAN_START_TIME,format}` 模板渲染。
- 支持 `${SCAN_START_TIME+15m,yyyyMMddHHmm}`、`${SCAN_START_TIME+1h,...}`、`${SCAN_START_TIME+1d,...}` 等偏移。
- 远程扫描匹配数为 0 时返回包含源类型、扫描目录、扫描文件数、正则和扫描时间的错误。
- `source.toml` 示例已使用安全占位配置，不包含真实账号密码。

## 配置文件

远程来源配置文件为 `source.toml`，示例：

```toml
[source]
type = "sftp"
download_dir = "./downloads"
remote_pattern = "/data/pm/${SCAN_START_TIME,yyyyMMdd}/EastCom_WIRELESS_PM_ENB_EUTRANCELL_V3.*_${SCAN_START_TIME,yyyyMMddHHmm}_${SCAN_START_TIME+15m,yyyyMMddHHmm}.*.csv.zip"
cache_retention_days = 7
connect_retry = 3
download_retry = 3
retry_interval_secs = 30
connect_timeout_secs = 30
read_timeout_secs = 300

[source.connection]
host = "127.0.0.1"
port = 22
username = "user"
password = "password"
```

真实环境部署时，只替换 `[source.connection]` 和 `remote_pattern`，不要把真实密码提交到仓库。

## 真实远程回归

真实 FTP/SFTP 环境已由用户完成手工回归验证。

仓库中不记录真实远程地址、用户名、密码和内部目录。回归记录只保留结论：

- 远程连接可用。
- 远程目录扫描可用。
- `SCAN_START_TIME` 模板渲染符合预期。
- 正则匹配可命中目标文件。
- 远程文件可下载到本地缓存目录。
- 下载后的文件可进入阶段 1 本地解析、TPD 聚合和输出包流程。

## 已知限制

- SFTP 当前只支持用户名密码认证，不支持密钥认证。
- FTP 使用 `suppaftp` 同步 API，当前不单独暴露主动/被动模式配置。
- 不限制远程递归深度，`remote_pattern` 应尽量缩小扫描目录范围。
- 不同远程目录下同名文件会下载到同一个本地文件名并互相覆盖。
- `remote_pattern` 是正则表达式，点号等字符需要按正则规则转义。
- XML 解析仍未实现，不属于阶段 2 范围。
