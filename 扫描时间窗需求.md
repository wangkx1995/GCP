# 扫描时间窗计算功能需求（ScanWindow）

> 周期性文件采集场景下，`scan_start_time` / `scan_end_time` 的计算规格。
> 对应 Java 原版：`com.boco.gcp.util.DateUtil.getScanScope(Date, long, long)` + `getDiff(String, int)`。
> 适用重构目标：Rust 版周期调度模块。

---

## 1. 功能名称

周期性文件采集的扫描时间窗计算（ScanWindow 计算）

## 2. 功能描述

系统对周期性文件采集策略，需要计算出"本次下发应采集哪一段数据"的时间窗（`scan_start_time` / `scan_end_time`）。该计算与"策略何时被触发"解耦：触发时间由 cron 表达式控制（含故意错开的宽限延迟），而数据时间窗由采集周期（period）对齐到固定周期网格得出。

## 3. 输入

| 参数 | 类型 | 说明 |
|---|---|---|
| `fire_time` | 时间戳(ms) | 本次 CheckPolicyThread 扫描到的当前分钟（已对齐到整分）也就是每次任务运行的时间 5/15 * * * | 
| `period` | 整数(秒) | 采集周期，如 900(15分)/3600(1时)/86400(1天)/604800(1周)/2592000(30天) |
| `delay_period` | 整数(秒) | 宽限延迟，采集触发相对数据窗关闭点的滞后量。默认值 0，单位秒，不允许负数。需要在采集策略表（`collection_strategy`）新增字段，页面可配置。时间计算按北京时间（UTC+8）处理。|

## 4. 输出

返回长度为 2 的长整型数组 `[scan_start_time, scan_end_time]`（毫秒），分别对应数据窗起点与终点；`period <= 0` 时返回 `null`（不采集）。

## 5. 处理逻辑

5.1 将 `fire_time` 的秒、毫秒置零，得到"当前整分时刻"。

5.2 按 `period` 大小分档对齐（详见第 6 节），核心动作是**把当前时刻向下 floor 到周期网格的整数边界**。

5.3 对齐得到的边界作为窗终点候选，再整体减去 `delay_period` 得到最终窗。**窗口宽度通常为一个完整周期 `period`**，`period == 2592000`（日历月）除外，此时窗宽由日历月自然边界决定：

- 一般情况（`period != 2592000`）：
  - `delay_period == 0`：终点 = 对齐边界，起点 = 对齐边界 − period；
  - `delay_period > 0`：终点 = 对齐边界 − delay，起点 = 对齐边界 − period − delay。
- 日历月分支（`period == 2592000`）：
  - 对齐边界 = 本月 1 号 00:00:00；
  - 终点 = 对齐边界 − delay；
  - 起点 = 上月 1 号 00:00:00 − delay。

## 6. 各周期档位对齐行为

| period | 对齐方式 |
|---|---|
| `>= 3600` 且非整值（如 2 小时） | 按周期毫秒（或折算分钟数）取模，向下 floor 到周期网格。例如 2 小时周期，`05:30` 对齐到 `04:00` |
| `== 3600`（1 小时） | 分钟置零 → 本小时整点（按北京时间） |
| `== 86400`（1 天） | 时、分置零 → 当天 00:00:00（按北京时间） |
| `== 604800`（1 周） | 对齐到**周一 00:00:00**（按北京时间） |
| `== 2592000`（30 天） | 对齐到**本月 1 号 00:00:00**；最终窗口为"上月 1 号 00:00:00"至"本月 1 号 00:00:00"（日历月，按北京时间；窗宽不一定等于 2592000 秒） |
| `< 3600`（如 900=15 分，兜底分支） | `对齐时间 = fire_time − ((fire_time + 8小时) % (period×1000))`。`+8h` 表示按北京时间（UTC+8）对齐周期网格，向下 floor 到周期边界 |

## 7. 边界与异常

- `period <= 0` → 返回 `null`，不触发采集。
- `delay_period < 0` → 非法输入；实现时统一按 0 处理（或返回错误，由调用方决定）。
- 亚小时周期（< 3600）一律走取模兜底分支，不进入整点/整天分支。
- `+8h` 偏移表示按北京时间（UTC+8）对齐周期网格，保证与本地日期时间一致。
- 日历不存在的日期（如 2 月 30 日）由日历对象自动滚到下月合法日，不抛异常。

## 8. 与 cron 触发的职责边界（重要）

- cron 表达式（如 `5/15 * * * *`）**只决定"线程何时醒来算一次"**，不参与数据窗计算；
- `scan_start_time` **永远由 period 周期网格算出**，因此即使在第 5、20、35、50 分触发，采集的数据窗起点仍落在 0/15/30/45 分的周期格上；
- cron 相对周期故意错开 5 分钟，是为文件落地预留宽限，不属于本功能计算范围。

## 9. 验收标准

1. 给定 `fire_time = 某日 05:00:00`、`period = 900`、`delay = 0`，输出窗应为 `[04:45:00, 05:00:00]`（终点为向下对齐的 15 分钟边界）。
2. 给定 `fire_time = 某日 05:00:00`、`period = 900`、`delay = 300`，输出窗应为 `[04:40:00, 04:55:00]`（整周期窗口，终点回退 5 分钟）。
3. `period = 3600` 时，任意 `fire_time` 的输出窗终点应为其所在小时的整点。
4. `period = 86400` 时，输出窗终点应为当天 00:00:00。
5. `period = 0` 时返回 `null`。
6. 不同 `fire_time` 落在同一周期格内时，计算出的 `scan_start_time` 必须一致（幂等性）。

## 10. Rust 实现要点

```rust
use chrono::{DateTime, FixedOffset, TimeZone, Timelike, Datelike};

/// 计算扫描时间窗 [start, end]（毫秒）
/// period / delay_period 单位：秒
fn get_scan_scope(fire_time_ms: i64, period_sec: i64, delay_sec: i64) -> Option<(i64, i64)> {
    if period_sec <= 0 { return None; }
    let delay_sec = delay_sec.max(0);

    // 1. 秒、毫秒置零 → 整分
    let fire_min = (fire_time_ms / 60_000) * 60_000;

    // 2. 按 period 分档对齐到周期边界
    let aligned = match period_sec {
        3600 => align_hour(fire_min),
        86400 => align_day(fire_min),
        604800 => align_week(fire_min),
        2592000 => align_month(fire_min),
        _ if period_sec >= 3600 => align_to_period_grid(fire_min, period_sec),
        _ => align_to_period(fire_min, period_sec),
    };

    // 3. delay 回退
    let delay_ms = delay_sec * 1000;
    let (end, start) = if period_sec == 2592000 {
        // 日历月分支：窗宽为上月 1 号到本月 1 号
        let start = prev_month_first_day_ms(aligned) - delay_ms;
        let end = aligned - delay_ms;
        (end, start)
    } else {
        // 一般分支：窗宽恒为 period
        let period_ms = period_sec * 1000;
        let end = aligned - delay_ms;
        let start = aligned - period_ms - delay_ms;
        (end, start)
    };

    Some((start, end))
}

/// 亚小时周期：向下 floor 到周期边界（按北京时间 UTC+8 对齐）
fn align_to_period(t: i64, period_sec: i64) -> i64 {
    let p = period_sec * 1000;
    let shift = 8 * 3600 * 1000;        // UTC+8 北京时间
    t - ((t + shift) % p)
}

/// 长周期（>=3600 非精确值）：直接按周期毫秒取模 floor
fn align_to_period_grid(t: i64, period_sec: i64) -> i64 {
    let p = period_sec * 1000;
    t - (t % p)
}

/// 1 小时：分钟置零（北京时间）
fn align_hour(t: i64) -> i64 { /* chrono truncate to hour in +08:00 */ }

/// 1 天：时分置零（北京时间）
fn align_day(t: i64) -> i64 { /* chrono truncate to day in +08:00 */ }

/// 1 周：对齐到周一 00:00:00（北京时间）
fn align_week(t: i64) -> i64 { /* chrono: previous Monday 00:00 +08:00 */ }

/// 30 天（日历月）：对齐到本月 1 号 00:00:00（北京时间）
fn align_month(t: i64) -> i64 { /* chrono: first day of current month +08:00 */ }

/// 返回本月 1 号对应的上月 1 号 00:00:00 毫秒时间戳（北京时间）
fn prev_month_first_day_ms(current_month_first_ms: i64) -> i64 { /* chrono in +08:00 */ }
```

> 注：
> - `3600 / 86400 / 604800 / 2592000` 建议用 `chrono` + 北京时间偏移（`FixedOffset::east_opt(8 * 3600)`）单独实现；
> - `>= 3600` 非整值复用按周期毫秒取模 floor；
> - `< 3600` 复用 `align_to_period` 取模兜底。
